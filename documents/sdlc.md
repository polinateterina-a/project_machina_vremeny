# ЗАДАЧИ ПО SDLC «Перекресток эпох»

## ЭТАП 1. ПЛАНИРОВАНИЕ

### 1.1. Инициация проекта

- Сформулировать концепцию проекта (миссия, ценности, проблематика)
- Определить целевую аудиторию (жители, туристы, молодёжь, исследователи)
- Сформулировать проблему и актуальность
- Собрать подтверждающие исследования и статистику
- Подготовить обоснование актуальности программными документами

### 1.2. Планирование ресурсов

- Определить команду проекта и распределить роли
- Оценить бюджет проекта (расходы на разработку, контент, хостинг)
- Составить календарный план (график работ по неделям)
- Определить технологический стек (Python/Flask, SQLite, Leaflet)

### 1.3. Риски и гипотезы

- Сформулировать продуктовые гипотезы (3–5 штук)
- Определить метрики успеха и KPI
- Проанализировать риски и способы их снижения

### 1.4. Документирование

- Подготовить паспорт проекта

## ЭТАП 2. АНАЛИЗ ТРЕБОВАНИЙ

### 2.1. Сбор требований

- Проанализировать аналоги
- Провести интервью с потенциальными пользователями (краеведы, туристы)
- Собрать требования от партнёров (музей, архив, библиотека)
- Изучить отзывы на существующие решения

### 2.2. Формализация требований

- Составить функциональные требования (что система должна делать)

- Определить требования к контенту (форматы фото, аудио, текстов)
- Составить пользовательские сценарии

### 2.3. Утверждение требований

## ЭТАП 3. ПРОЕКТИРОВАНИЕ

### 3.1. Архитектурное проектирование

- Разработать схему архитектуры информационной системы
- Определить компоненты системы (фронтенд, бэкенд, БД, хранилище)
- Спроектировать взаимодействие между компонентами (API)
- Выбрать окончательный технологический стек

### 3.2. Проектирование базы данных

- Создать ER-диаграмму (таблицы: города, экскурсии, локации)
- Определить атрибуты каждой таблицы и связи
- Спроектировать индексы для ускорения поиска
- Создать скрипт инициализации базы данных (init_db.py)

### 3.3. Проектирование API

- Определить все эндпоинты (GET, POST)
- Описать формат запросов и ответов (JSON)
- Спроектировать обработку ошибок и статусы ответов
- Составить спецификацию API (в формате OpenAPI или текстовом)

### 3.4. Проектирование интерфейса (UI/UX)

- Создать вайрфреймы (главная страница, карточка локации, админ-панель)
- Разработать дизайн-макеты в Figma
- Спроектировать адаптивность для мобильных устройств
- Создать кликабельный прототип для тестирования

### 3.5. Проектирование контента

- Составить список локаций (4 точки маршрута)
- Определить источники контента (музеи, архивы, частные коллекции)
- Написать шаблоны текстов для локаций
- Создать сценарии звуковых реконструкций

## ЭТАП 4. РАЗРАБОТКА

- Установить Git и инициализировать репозиторий

### 4.2. Создание структуры проекта
